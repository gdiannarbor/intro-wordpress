"Local" Installation

In the world of web developerment, running a website "locally" means running it on your personal computer.
(This is in contrast to hosting a website on a remote server, accessible only via the internet.)

Local environments are important because they allow us to work directly with our code and make changes more easily, faster, and without any internet connection at all.

If you want to design or develop for WordPress,
it's wise to work on a local install.

Local WordPress Installation

So what do we need to get WordPress running locally?

  • An environment on your computer with all the tools that WordPress needs to function.
  • A web browser. WordPress is configured and content is managed via the browser.

Setting Up a WordPress-Ready Development Environment

When we say we need an "environment" with all the tools that WordPress needs to function, we're talking about:

  • Apache
    An open source web server that processes web requests.
  • MySQL
    An open source database for storing content and other data.
  • PHP
    An open source programming language for building out websites.

We'll be using Vagrant to setup this local environment.

Vagrant & Virtualization

We could install Apache, MySQL, PHP, WordPress, and all of their depencies one-by-one on our machines - but this is time consuming, delicate, and has a steep learning curve.

Vagrant is a tool that allows us to quickly, and repeatedly, create a "virtual" environment - a complete operating system, running within another operating system - that includes all the tools we need.

"It allows you, or someone else, to specify in plain text how the machineâ€™s virtual hardware should be configured and what should be installed on it. It also makes it insanely easy to get the code on the server. You check out your project, type vagrant up and start work." [source]

Varying Vagrant Vagrants

Varying Vagrant Vagrants is an open source Vagrant configuration focused on WordPress development.

Setting up VVV creates your virtual environment along with all the tools you'll need to run WordPress: Apache, MySQL, PHP, WordPress itself, and all of the libraries these systems depend on.

Optional Add-ons for VVV

There are a couple of additional resources that could
make your life easier. I use the following:

  • Vagrant Manager - allows me to make and destory virtual machines as I need them.
  • Variable VV - This makes it easier to create and destory new local Wordpress installations.
  • VVV-Dashboard - A nice display of all your VVV sites with links to their Wordpress dashboards.

You're all set up!
